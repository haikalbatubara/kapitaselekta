Judul: Segmentasi Pelanggan PT. Bank Sumut Berdasarkan Data Transaksi M-Banking Menggunakan K-Means Clustering

Ringkasan (Abstract singkat)

Penelitian ini menerapkan algoritma machine learning K-Means Clustering untuk segmentasi pelanggan PT. Bank Sumut berdasarkan data transaksi M-Banking. Proses melibatkan analisis kebutuhan, perancangan alur, validasi hasil clustering, serta pengelolaan proyek analisis data.

  • SWEBOK: Software Requirements, Software Design, Software Testing, Software Engineering Management

Tools & Pendekatan

  • Bahasa: Python
  • Metode: K-Means Clustering
  • Dataset: Data transaksi M-Banking
  • Pendukung: Cloud storage, dataset publik

Prioritas Internet — 10 Kekuatan Pendatar Dunia

  • Outsourcing: memanfaatkan layanan cloud & library publik.
  • Offshoring: kolaborasi riset dan data lintas negara.
  • Open-sourcing: kontribusi & akses software terbuka.
  • Insourcing: optimalisasi data internal organisasi.
  • Supply-chaining: integrasi layanan perbankan dengan digital channel.
  • In-forming: memanfaatkan Google & data publik untuk insight.
  • Workflow software: pipeline otomatis untuk analisis clustering.
  • Uploading: berbagi kode & laporan melalui platform online.
  • Wireless: pemanfaatan mobile banking untuk data transaksi.
  • Digital content: publikasi hasil riset ke platform akademik.

Pemahaman Open Source Philosophy

Open Source Philosophy menekankan transparansi, kolaborasi, dan kebebasan dalam mengembangkan software. Dalam konteks penelitian ini, prinsip open source dipakai dengan memanfaatkan library Python terbuka (seperti Scikit-learn), serta berbagi kode agar dapat digunakan kembali oleh peneliti lain.

Prioritas saya: menerapkan filosofi sharing & collaboration, karena dengan berbagi kode dan dataset, penelitian bisa lebih cepat berkembang dan menghasilkan solusi yang berguna bagi industri perbankan.

Analisis SWOT & Porter

SWOT (Ringkasan)

  • Strengths: Minat & pemahaman machine learning, terbiasa analisis data keuangan.
  • Weaknesses: Implementasi real-time terbatas, perlu pendalaman validasi model.
  • Opportunities: Industri perbankan butuh segmentasi pelanggan, data M-Banking kaya.
  • Threats: Risiko keamanan data nasabah, akurasi bergantung kualitas data.

Analisis Porter

  • Ancaman pendatang baru: tinggi (banyak riset ML).
  • Kekuatan pemasok: data transaksi sebagai aset utama.
  • Kekuatan pembeli: bank sangat butuh insight pelanggan.
  • Substitusi: metode lain (Decision Tree, Random Forest, Deep Learning).
  • Persaingan: sedang (riset serupa ada, tapi fokus Bank Sumut membedakan).
